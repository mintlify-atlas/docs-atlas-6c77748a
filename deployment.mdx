### Dedicated VPS

For maximum performance and control, host on a dedicated VPS. Recommended providers:

- **[Crunchbits](https://crunchbits.com/)** — current official hosting provider for Holy Unblocker LTS
- **[Oracle Cloud](https://www.oracle.com/cloud)** — free tier available with paid upgrade options
- **[Azure](https://azure.microsoft.com)** — free and paid tiers available

When using a VPS, follow the [VPS setup guide](/self-hosting/vps-setup) and set `production` to `true` in `config.json` to enable PM2 process management.

## GitHub Codespaces

GitHub Codespaces provides a cloud development environment you can use from any browser, including Chromebooks. This is suitable for development and testing, not long-term production hosting.

<Steps>
  <Step title="Fork and star the repository">
    Fork the [Holy-Unblocker repository](https://github.com/QuiteAFancyEmerald/Holy-Unblocker) to your GitHub account, then star it.
  </Step>

  <Step title="Open Codespaces">
    Go to [https://github.com/codespaces](https://github.com/codespaces). Make sure you are signed into your GitHub account.
  </Step>

  <Step title="Create a new codespace">
    Click **New codespace** and configure it as follows:

    - **Repository:** `[YOUR_USERNAME]/Holy-Unblocker`
    - **Branch:** `master`
    - **Dev container configuration:** Holy Unblocker LTS

    Click **Create codespace** and wait for the container to finish setting up.
  </Step>

  <Step title="Install and start">
    In the terminal that opens inside the codespace, run:

    ```bash
    npm run fresh-install
    npm start
    ```
  </Step>

  <Step title="Make the port public">
    When the application popup appears, click **Make public**. Then open the **Ports** tab in VS Code to find the forwarded URL for port `8080`.
  </Step>
</Steps>

## Environment requirements

| Requirement | Value |
|-------------|-------|
| Node.js version | `20.x` |
| Default port | `8080` |
| Production branch | `v6.9.5_production` |
| Docker image | `quiteafancyemerald/holy-unblocker:6.9.5` |

The port can be changed in `ecosystem.config.js` by setting the `PORT` environment variable under `env` or `env_production`.

## Docker

You can pull and run the official Docker image directly:

```bash
docker pull quiteafancyemerald/holy-unblocker:6.9.5
docker run -p 8080:8080 quiteafancyemerald/holy-unblocker:6.9.5
```

For production deployments via Dockerfile (e.g. on Koyeb), use the `v6.9.5_production` branch — it includes a `Dockerfile` configured for port `8080`.
